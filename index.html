DP Payments — Search badges & ranking boost

Monolith (Search App + Search Bridge) · Marketplace payment search index · DB snapshot 2026-04-13 · Countries: BR, ES, IT, MX, PL

What this feature is

Patient-facing search cards can show trust signals tied to DP Payments, and the same signals feed optional ranking factors when sorting by “Recommended”.

  • “Secure payments” badge — shown per address when online payments are flagged as enabled in the payment search index (payments_enabled → has_badge / hasBadge in the search payload). Template: SearchBridgeBundle/Resources/views/SearchApp/info.block.html.twig.
  • “Renowned specialist / facility” line — driven by frequently_paid (has_frequent_badge / hasFrequentBadge). Copy differs for doctor vs facility. Template: paymentsRenownBadge.block.html.twig.
  • Ranking — AddressPaymentsDocumentBuilder writes Elasticsearch-style factors.payments_* fields used by Search Bridge factors (PaymentsHasBadgeFactor, PaymentsFrequentlyPaidFactor, PaymentsTierFactor).
Product caveat
The “secure payments” badge is determined at address / account level in the index, not per priced service. A known gap exists where the badge may show even when no service can actually be paid online (e.g. all “price from”). Eligibility at booking uses stricter checks (GetMarketplacePaymentFlow).

Country & entity behaviour

Gate: GATE_PAYMENTS_BADGES_ON_SEARCH_RESULTS in GateKeeper.class.php — enabled only for br, es, it, mx, pl. Outside these markets, payment badge factors return an empty definition and payment index documents for search are not built.

Doctors

  • Parent row: crm_marketplace_payment_search_index.doctor_id
  • Address row: crm_marketplace_payment_search_index_address.address_id
  • Indexing also considers rows with promoted_doctor_id when resolving which doctors to update

Facilities

  • Parent row: crm_marketplace_payment_search_index.facility_id
  • Address row: facility_address_id
  • UI uses facility-specific translation keys for the “renowned” line

There is no separate country list for doctors vs facilities: the same five countries apply; the split is entity type and how the address key is stored.
